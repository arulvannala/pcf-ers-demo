Welcome to the Sample Vmware Tanzu Application Service. If you're here it means you successfully deploy the application. Learn more.

To get here we simply used the CF CLI PUSH command to deploy your application or artifact. We often use a manifest yaml file to encapsulate the command option. The application binaries and metadata are upload, an application is created along with a route and associated with your artifact and metadata Learn more.

tas Elastic Runtime run applications in containers and part of the tas staging process is to create an application container image called droplet.

Note: tas also supports running applications from a docker image. In that case, the buildpack is not used during the staging process.
